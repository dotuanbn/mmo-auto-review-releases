## 2) Phạm vi vận hành hiện tại (quan trọng)

- Hệ thống đang đi theo hướng **compliance-first**.
- Luồng review hiện tại có cơ chế **manual confirm trước submit** (không auto-submit toàn cục mặc định).
- Traffic/Web browsing automation vẫn chạy agentic + heuristic fallback, có logging realtime/report.

## 5) Data root và portable mode

### Cơ chế hiện tại

- `DataRootService` đang dùng chiến lược hybrid để tránh mất dữ liệu khi build lại:
  - Mặc định packaged: dùng root ổn định ở `%APPDATA%/MMO Auto Review`
  - Chỉ bật portable root cạnh EXE (`<EXE_DIR>/data`) khi có:
    - env `MMO_PORTABLE_MODE=1`, hoặc
    - file marker `portable.mode` cạnh EXE
- File liên quan:
  - `src/main/services/DataRootService.ts`
  - `src/main/index.ts` (bootstrap set userData path trước khi init DB)

### Cấu trúc dữ liệu runtime phổ biến

- DB: `<dataRoot>/data/mmo-review.db`
- Logs proxy/runtime: `<dataRoot>/proxy-debug.log`
- Profiles trình duyệt:
  - `<dataRoot>/profiles/*` (BrowserService generic)
  - `<dataRoot>/traffic_profiles/*` (per-account traffic profile)

### Migration từ legacy

- IPC hỗ trợ:
  - `data:getRoot`
  - `data:detectLegacy`
  - `data:migrateLegacy`
- File:
  - `src/main/ipc/data.ts`

### Auto-seed dữ liệu khi đổi storage mode

- Khi đổi từ mode cũ sang mode mới, nếu target root đang rỗng thì hệ thống tự seed dữ liệu từ:
  1. default Electron `userData` cũ
  2. portable root cũ `<EXE_DIR>/data`

Mục tiêu: không mất dữ liệu khi chuyển mode hoặc khi build lại thư mục `release/win-unpacked`.
